# Анализ негативных отзывов клиентов Т-Банка

## Описание проекта
Данный проект представляет собой исследовательский анализ данных (EDA) отзывов клиентов Т-Банка. Целью работы является выявление ключевых проблемных зон в продуктах и сервисах банка на основе анализа негативных отзывов, а также сегментация этих проблем по направлениям бизнеса для выработки рекомендаций по улучшению клиентского опыта.

## Постановка задачи
Перед анализом стояла задача понять, какие именно аспекты работы банка вызывают недовольство клиентов. Необходимо было не просто посчитать количество негатива, а провести глубокую декомпозицию:
*   В каких бизнес-линиях (кредиты, карты, инвестиции и др.) чаще всего возникают проблемы?
*   Какие конкретные темы и подтемы (например, "мобильное приложение" -> "ошибка входа") являются самыми болезненными для клиентов?
*   Существуют ли различия в восприятии проблем у разных сегментов клиентов (возраст, статус VIP, новый клиент)?

На основе полученных инсайтов требовалось сформулировать продуктовые гипотезы, направленные на снижение уровня недовольства и улучшение ключевых метрик удовлетворенности.

## Описание данных
Датасет содержит отзывы действующих клиентов Т-Банка о качестве продуктов, услуг и клиентского сервиса. Данные собраны с внешних платформ (таких как Banki.ru, Sravni.ru) и социальных сетей (ВКонтакте, Одноклассники и др.) за период с января по ноябрь 2025 года.

Каждый отзыв отражает реальный опыт клиента — от использования мобильного приложения до обращения в поддержку. Все обращения, содержащие жалобы, проходят модерацию и направляются специалистам для решения вопроса.

### Описание переменных

## Методология анализа

Анализ проводился в несколько этапов, с фокусом на сегментацию негативных отзывов.

1.  **Подготовка и очистка данных:**
    *   Обработка пропусков в оценках и тональности.
    *   Приведение текстовых полей к единому формату.

2.  **Расчет агрегированных метрик:**
    *   Для оценки общего уровня лояльности использовался подход, близкий к **NPS**: позитивными считались оценки 4–5 (и нейтральные/позитивные тексты без оценки), негативными — оценки 1–3 (и негативные тексты без оценки).
    *   Рассчитывался средний **CSAT (Customer Satisfaction Score)** в различных разрезах.

3.  **Анализ структуры негатива:**
    *   **Топ проблем:** Построены рейтинги самых частых `reason`, `review_theme` и `subtheme` среди негативных отзывов.
    *   **Декомпозиция по продуктам:** Для каждой бизнес-линии (`business_line`) и продукта (`product`) определены доли негатива и самые частые темы жалоб.
    *   **Сегментный анализ:** Сравнение структуры негатива между различными группами клиентов (новые vs. старые, VIP vs. обычные, разные возрастные группы).

4.  **Текстовый анализ (поверхностный):**
    *   Анализ частотности слов и словосочетаний в текстах негативных отзывов для подтверждения и детализации выводов, полученных из тематик.

## Результаты

В ходе анализа были выявлены ключевые зоны роста:

*   **Основные драйверы негатива:** Определен перечень самых частых причин недовольства (например, проблемы с работой мобильного приложения, длительное обслуживание, невыгодные условия по отдельным продуктам).
*   **Проблемные бизнес-линии:** Выявлены направления, требующие первоочередного внимания, на основе доли негативных отзывов и их влияния на общий CSAT.
*   **Портрет недовольного клиента:** Обнаружены различия в восприятии проблем разными сегментами (например, новые клиенты чаще жалуются на сложность интерфейса, а сеньоры — на качество обслуживания в отделениях).

## Продуктовые гипотезы

На основе проведенного анализа были выдвинуты две продуктовые гипотезы, направленные на снижение доли негативных обращений:

1.  **Гипотеза 1 (Улучшение онбординга):**
    *   **Что обнаружено:** Среди новых клиентов (`new_flg = 1`) высока доля негатива, связанного с темами "мобильное приложение" и "активация продукта".
    *   **Гипотеза:** Внедрение интерактивного онбординга (подсказок и гайдов) при первом входе в мобильное приложение для новых клиентов снизит долю негативных отзывов по этим темам на **X%** в течение первых 3 месяцев использования.

2.  **Гипотеза 2 (Приоритизация поддержки):**
    *   **Что обнаружено:** Отзывы VIP-клиентов (`influencer_flg = 1`) с высокой оценкой (4-5) часто содержат нейтральные или негативные тексты, связанные с долгим ожиданием ответа в поддержке.
    *   **Гипотеза:** Выделение отдельной очереди (приоритетной линии) для VIP-клиентов в каналах поддержки повысит их удовлетворенность, что отразится в росте средней оценки и CSAT-скорра в этом сегменте.

### План проверки гипотезы (на примере Гипотезы 2)

Для проверки гипотезы о приоритетной поддержке для VIP-клиентов предлагается дизайн A/B-теста:

*   **Единица эксперимента:** Клиент (VIP-сегмент).
*   **Метрика:** CSAT-скорр, рассчитанный по следующему обращению в поддержку.
*   **Группы:**
    *   **Контрольная группа:** VIP-клиенты обслуживаются в общем потоке.
    *   **Экспериментальная группа:** VIP-клиенты попадают в отдельную, приоритетную очередь с гарантированным временем ответа < 1 минуты.
*   **Ожидаемый результат:** Статистически значимое (p-value < 0.05) увеличение среднего CSAT-скорра в экспериментальной группе по сравнению с контрольной.
