# Анализ негативных отзывов клиентов Т-Банка

## Описание проекта
Данный проект представляет собой исследовательский анализ данных (EDA) отзывов клиентов Т-Банка. Целью работы является выявление ключевых проблемных зон в продуктах и сервисах банка на основе анализа негативных отзывов, а также сегментация этих проблем по направлениям бизнеса для выработки рекомендаций по улучшению клиентского опыта.

## Постановка задачи
Перед анализом стояла задача понять, какие именно аспекты работы банка вызывают недовольство клиентов. Необходимо было не просто посчитать количество негатива, а провести декомпозицию:
*   В каких бизнес-линиях (кредиты, карты, инвестиции и др.) чаще всего возникают проблемы?
*   Какие конкретные темы и подтемы (например, "мобильное приложение" -> "ошибка входа") являются самыми болезненными для клиентов?

На основе полученных инсайтов требовалось сформулировать продуктовые гипотезы и описать, как их можно протестировать.

## Описание данных
Датасет содержит отзывы действующих клиентов Т-Банка о качестве продуктов, услуг и клиентского сервиса. Данные собраны с внешних платформ (таких как Banki.ru, Sravni.ru) и социальных сетей (ВКонтакте, Одноклассники и др.) за период с января по ноябрь 2025 года.

Каждый отзыв отражает реальный опыт клиента — от использования мобильного приложения до обращения в поддержку. Все обращения, содержащие жалобы, проходят модерацию и направляются специалистам для решения вопроса.

### Описание переменных

## Методология анализа

Анализ проводился в несколько этапов, с фокусом на сегментацию негативных отзывов.

1.  **Подготовка и знакомство с данными:**
    *   Построение распределений признаков.
    *   Сегментация категориальных переменных.

2.  **Расчет агрегированных метрик:**
    *   Для оценки общего уровня лояльности использовался подход, близкий к **NPS**: позитивными считались оценки 4–5 (и нейтральные/позитивные тексты без оценки), негативными — оценки 1–3 (и негативные тексты без оценки).

3.  **Анализ структуры негатива:**
    *   **Топ проблем:** Построены рейтинги самых частых `reason`, `review_theme` и `subtheme` среди негативных отзывов.
    *   **Декомпозиция по продуктам:** Для каждой бизнес-линии (`business_line`) определены доли негатива и самые частые темы жалоб.

## Результаты

В ходе анализа были выявлены ключевые зоны роста:

*   **Основные драйверы негатива:** Определен перечень самых частых причин недовольства.

## Продуктовые гипотезы

На основе проведенного анализа были выдвинуты две продуктовые гипотезы, направленные на снижение доли негативных обращений:

1.  **Гипотеза 1:**
    *   **Что обнаружено:** У клиентов возникают вопросы, связанные с кэшбеком (когда начислят, сколько начислят и пр.).
    *   **Гипотеза:** Добавление на экране карты интерактивного счетчика кэшбека (по этой карте вы получите X рублей, Y уже начислено; пункт “как считаем?”) уменьшит долю вопросов по кэшбеку.

2.  **Гипотеза 2 (Приоритизация поддержки):**
    *   **Что обнаружено:** Большинство негативных отзывов связаны с блокировкой/разблокировкой карты.
    *   **Гипотеза:** Размещение в мобильном приложении трекер блокировки с конкретными шагами для разблокировки (“карта приостановлена из-за операции …” – “для разблокировки подтвердите эти 3 транзакции” – “после подтверждения карта разблокируется автоматически через 2 минуты”), то количество отзывов на тему блокировки/разблокировки карты уменьшится.

### План проверки гипотезы (на примере Гипотезы 2)

Для проверки гипотезы о блокировки/разблокировки карты предлагается дизайн A/B-теста:

*   **Единица эксперимента:** Клиент, использующий дебетовую карту и получающий в течение месяца на нее не менее Z рублей.
*   **Метрика:** доля пользователей, обращающихся в поддержку по поводу разблокировки/блокировки карты.
*   **Группы:**
    *   **Контрольная группа:** текущий опыт.
    *   **Экспериментальная группа:** опыт с трекером.
